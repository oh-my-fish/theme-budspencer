    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-budspencer-theme" class="anchor" aria-hidden="true" href="#budspencer-theme"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Budspencer theme</h1>
<p>A theme for nerds, makes fish even more powerful. It's equipped with a hell of a
lot of nice functions and key bindings to speed up your workflow.</p>
<p>The philosophy behind budspencer theme:</p>
<ul>
<li>Delimit the prompt lines clearly to the output.</li>
<li>Produce output only if it contains relevant information.</li>
<li>Show useful notifications within the prompt.</li>
<li>Equip the prompt with toggle commands in order to change its style
interactively.</li>
<li>Store sessions in order to make frequently used commands quickly accessible.</li>
<li>Make navigation within the directory tree as easy as possible using
bookmarks and a directory history without duplicates.</li>
<li>Make histories quickly accessible with menus.</li>
<li>Don't stress your eyes. Use different colors in dark and bright environments.</li>
</ul>
<h2><a id="user-content-demonstration-video" class="anchor" aria-hidden="true" href="#demonstration-video"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Demonstration video</h2>
<p><a href="http://vimeo.com/105546618" rel="nofollow"><img src="https://raw.githubusercontent.com/tannhuber/media/master/budspencer.jpg" alt="video" style="max-width:100%;"></a></p>
<h2><a id="user-content-requirements" class="anchor" aria-hidden="true" href="#requirements"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Requirements</h2>
<p>You need an up-to-date fish shell version from <a href="https://github.com/fish-shell/fish-shell.git">github</a> or a
<a href="https://github.com/fish-shell/fish-shell/wiki/Nightly-builds">nightly build</a>.</p>
<p>Non standard tools:</p>
<ul>
<li>xsel</li>
<li>wmctrl</li>
<li>git</li>
<li><a href="http://taskwarrior.org/" rel="nofollow">taskwarrior</a> (optional)</li>
<li><a href="http://www.roaringpenguin.com/products/remind" rel="nofollow">remind</a> (optional)</li>
</ul>
<h3><a id="user-content-note-for-os-x-users" class="anchor" aria-hidden="true" href="#note-for-os-x-users"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Note for OS X users</h3>
<p>The theme uses advanced <code>sed</code> and <code>expr</code> features. The OS X tools have
restricted functionality.</p>
<p>Try <code>expr</code> from <code>coreutils</code> package if you see <code>expr: syntax error</code> in
budspencer prompt.</p>
<p>Install advanced <code>sed</code> and <code>expr</code> versions:</p>
<pre><code>$ brew install coreutils gnu-sed
</code></pre>
<p>Add this to your <code>~/.config/fish/config.fish</code>:</p>
<pre><code>if test (uname -s) = "Darwin"
  set -gx PATH /usr/local/opt/coreutils/libexec/gnubin $PATH
  set -gx PATH /usr/local/opt/gnu-sed/libexec/gnubin $PATH
end
</code></pre>
<h2><a id="user-content-configuration" class="anchor" aria-hidden="true" href="#configuration"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Configuration</h2>
<h3><a id="user-content-general-configuration" class="anchor" aria-hidden="true" href="#general-configuration"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>General configuration</h3>
<p>The theme behaves similar to vim's airline/powerline plugins. Thus, it needs a
<a href="https://github.com/Lokaltog/powerline-fonts">powerline font</a>.
It's also crucial to have vi mode enabled. Thus, execute</p>
<pre><code>set -U fish_key_bindings fish_vi_key_bindings
</code></pre>
<p>before enabling the theme.</p>
<h3><a id="user-content-colors" class="anchor" aria-hidden="true" href="#colors"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Colors</h3>
<p>Redefine <code>$budspencer_colors</code> with the <code>set</code> command in order to change the
colors of the prompt.</p>
<p>Example:</p>
<pre><code>set budspencer_colors 000000 333333 666666 ffffff ffff00 ff6600 ff0000 ff0033 3300ff 0000ff 00ffff
00ff00
</code></pre>
<p>will result in a prompt like this:

<p>The function <code>c</code> drops down a menu showing the command history as enumerated list similarly to
the <code>d</code> function. Selections are also pasted into the X clipboard. It's possible

<h2><a id="user-content-bookmarks" class="anchor" aria-hidden="true" href="#bookmarks"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Bookmarks</h2>
<p>A bookmark can be created with <code>mark</code>. It can be removed with <code>unmark</code>.</p>
<p><code>unmark</code> can take an argument to remove a specific bookmark in the bookmark list.
Simply add the number of the bookmark you would like to remove as the argument.</p>
<p>Bookmarks are universal and thus persistant.
A new shell automatically changes working directory on startup to newest bookmark.
<code>m</code> is a function that drops down a menu showing the bookmarks as enumerated list
equivalently to <code>d</code>.</p>
<p>The following shortcuts need vi mode:</p>
<ul>
<li>Create a bookmark for present working directory with <code>m</code> in NORMAL mode.</li>
<li>Remove a bookmark for present working directory with <code>M</code> in NORMAL mode.</li>
</ul>
<p>By default the shell will change to the directory of the latest bookmark on a new session.
This can be disabled by adding the following line:</p>
<pre><code>set -U budspencer_no_cd_bookmark
</code></pre>

<p>Shell session can be stored with <code>s &lt;session name&gt;</code>. If a session with <code>session name</code> already exists, the session with this name will be attached. If a session with
<code>session name</code> is already active within another terminal, this terminal will be
focussed.</p>
<p>A list of available sessions can be shown with <code>s</code>.</p>
<p>A session can be erased with <code>s -e &lt;session name&gt;</code>.</p>
<p>Type <code>s -d</code> to detach current session.</p>
